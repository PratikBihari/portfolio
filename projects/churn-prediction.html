Customer Churn Prediction

Machine learning model to identify at-risk customers and reduce churn

Completed: December 2022
Client: TelecomX
Duration: 3 months
Back to Projects

Project Overview

TelecomX, a telecommunications provider with over 5 million subscribers, was experiencing a concerning increase in customer churn rate. The company was losing valuable customers to competitors without understanding the underlying reasons or having the ability to proactively identify at-risk customers.

The client needed a predictive solution that could identify customers likely to churn before they actually did, allowing the retention team to implement targeted intervention strategies and reduce overall churn rate.

Technical Approach

I developed a comprehensive churn prediction solution using machine learning techniques, following these key steps:

Data Collection & Preparation

I gathered and integrated data from multiple sources, including:

  • Customer demographics and account information
  • Service usage patterns (calls, data, messaging)
  • Billing and payment history
  • Customer service interactions
  • Network quality metrics

After extensive data cleaning and feature engineering, I created a rich dataset with over 50 potential predictors of churn behavior.

Exploratory Data Analysis

I conducted thorough exploratory analysis to understand the relationships between various factors and churn behavior. Key insights included:

  • Customers with frequent billing issues were 3x more likely to churn
  • Service outages increased churn probability by 45%
  • Contract type was a strong predictor (month-to-month contracts had 5x higher churn)
  • Customer tenure showed a non-linear relationship with churn probability

Model Development

I evaluated several machine learning algorithms, including:

  • Logistic Regression
  • Random Forest
  • Gradient Boosting
  • Support Vector Machines
  • Neural Networks

After comparing their performance using cross-validation, Random Forest emerged as the most effective model, providing the best balance between accuracy, interpretability, and computational efficiency.

Challenges & Solutions

Throughout the project, several challenges emerged that required innovative solutions:

Class Imbalance

The dataset had a significant class imbalance, with churned customers representing only about 15% of the total. To address this, I implemented:

  • SMOTE (Synthetic Minority Over-sampling Technique) to generate synthetic examples of the minority class
  • Class weighting in the model to penalize misclassification of the minority class more heavily
  • Threshold optimization to balance precision and recall

Feature Selection

With over 50 potential predictors, there was a risk of overfitting and computational inefficiency. I implemented a robust feature selection process using:

  • Recursive Feature Elimination with Cross-Validation (RFECV)
  • Permutation importance analysis
  • Domain knowledge to validate statistical findings

Model Interpretability

While complex models like Random Forest can achieve high accuracy, they can be difficult to interpret. To make the model more transparent and actionable, I:

  • Implemented SHAP (SHapley Additive exPlanations) values to explain individual predictions
  • Created partial dependence plots to visualize the relationships between features and churn probability
  • Developed a simplified rule-based system derived from the model for quick reference by retention agents
